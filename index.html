OpenAGI 101 - Step 1

TL;DR:

  • You DO NOT need a server to use AI in your web page.
  • No python allowed anywhere. Everything is and will be done in normal JavaScript.
  • Free models on Groq work a lot faster than those charging money for slowness.
  • You DO NOT need to install anything, application built in this step is allready running below.
  • You can use AI models for free. No credit card required: https://console.groq.com/keys

Live application

100% done in JavaScript, using Preact and calling AI API directly with simple fetch calls.

No python, no server, no backend, no database, no paying, no nonesense.

Just JavaScript, 100% client side, running below without you needing to install anything.

After entering your Groq API key, select a model in the dropdown. Start with the latest llama-3.2 models.
The size of the model will affect your generate speed, 1B is fast but less smart than 3B or even 90B which is a lot smarter but also slower.

When you Submit, the AI will generate a response based on the prompt you entered. When "Loading..." is done, scroll to the bottom of the application to see it.

API call is configured to return JSON because text in this shape makes a lot more sense in real programming.
JSON will be parsed to generate data for our application code in later steps of our tutorial.

There is zero system prompt added, you need to specify JSON shape yourself with a simple description "what" you want "where" in the response.
Defining JSON shape is literally WYAIWYG: "what you ask for is what you get". Garbage in garbage out.
Asking for "use JSON format" is good for start, but not enough as it is unstable in the long run. Keys might change names or locations.
You will not know on which names to rely to find what you asked for when programming. Be specific on names and shapes (array, object, number, etc.).

Compare the quality of responses using different models. Submit again to use the new model with the same prompt.
